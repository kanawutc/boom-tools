/work-with
Communication

Persistent cross-oracle collaboration with synchronic scoring and party system

Description

Establish a persistent collaborative session with another oracle. Work-with tracks the collaboration history, scores synchronic alignment between oracles (how often they think alike, complement each other, or diverge productively), and maintains a "party" concept — oracles grouped into a working team with shared context. Collaboration that grows smarter over time.

Usage

/work-with <oracle> /work-with pimquin

Modes

Default

Open or resume collaboration session with target oracle

/work-with --party

Show current party — all oracles in active collaboration group

/work-with --score <oracle>

Show synchronic score history with a specific oracle

/work-with --add <oracle>

Add another oracle to the current party

/work-with --leave

Exit current party, archive collaboration history

/work-with --history

Show full collaboration history across all oracle partners

วิธีทำงาน

1

Resolve oracle identity from fleet or federation registry

2

Load existing collaboration history (if any)

3

Establish shared context channel via maw-js

4

Begin session — decisions logged with synchronic score

5

Update party state and score on session end

Output

Collaboration session state, synchronic score (0-100), party roster, shared context log, session history

Synchronic Score

Tracks how well two oracles collaborate: 90+ = deep resonance (same wavelength), 70-89 = productive complement (different but aligned), 50-69 = creative tension (healthy friction), below 50 = misalignment (needs review)

ใช้เมื่อไหร่?

ทำงาน cross-oracle project, ต้องการ track collaboration quality, หรือ build party สำหรับ multi-oracle mission

ใช้เมื่อไหร่ไม่ได้?

ไม่ใช่ real-time pair session — ใช้ /warp + /wormhole สำหรับ live interaction ส่วน /work-with เป็น persistent collaboration layer
