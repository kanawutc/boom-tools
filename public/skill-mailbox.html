/mailbox
Communication

Persistent agent mailbox — store findings, standing orders, and context for team agents across sessions

Description

A persistent asynchronous mailbox for agent-to-agent communication that survives across Claude sessions. Store findings, standing orders, context snapshots, and task results that team agents can read at their next session start. Solves the "context loss between sessions" problem in multi-oracle collaboration.

Usage

/mailbox /mailbox scout /mailbox --standing-orders

Modes

Default (read)

Show all pending messages in current oracle's mailbox

/mailbox <agent>

Read mailbox for a specific agent (e.g. /mailbox scout)

--standing-orders

Show and manage persistent standing orders for all agents

--send <agent> "msg"

Drop a message into another agent's mailbox

--clear <agent>

Clear read messages from an agent's mailbox

--archive

Move all read messages to archive, keep standing orders

วิธีทำงาน

1

Read mailbox files from ψ/inbox/ directory

2

Filter by agent name or show all pending

3

Display messages with sender, timestamp, priority

4

Mark messages as read (or queue for reply)

5

Surface standing orders at top — always visible

Output

Mailbox view: pending messages grouped by sender, standing orders pinned at top, archive status

ใช้เมื่อไหร่?

เมื่อ session เริ่ม (check ว่ามีงานรอ), ส่ง finding ให้ agent อื่น, set standing orders สำหรับ recurring tasks

ใช้เมื่อไหร่ไม่ได้?

ไม่ใช่ real-time chat — mailbox เป็น async เท่านั้น สำหรับ live communication ใช้ maw hey หรือ /talk-to แทน
