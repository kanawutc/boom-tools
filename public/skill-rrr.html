Core v3.5.2 G-SKLL

Session retrospective — AI diary + lessons learned + honest feedback

Usage

/rrr # Quick retro (default) /rrr --detail # Full template with all sections /rrr --dig # Reconstruct timeline from session .jsonl /rrr --deep # 5 parallel agents deep analysis

4 Modes

/rrr

Default — Quick retro. Main agent only. สรุป session, timeline, files, AI diary 150+ words, honest feedback 3 จุด, lessons learned

/rrr --detail

Full template — เพิ่ม Key Code Changes, Architecture Decisions, What Went Well/Could Improve, Blockers, Metrics

/rrr --dig

Session goldminer — ดึง timeline จาก .jsonl file จริง ได้ข้อมูลละเอียดกว่า จับ gap + idle ได้

/rrr --deep

5 parallel agents — วิเคราะห์ลึก ใช้ 5 agents สำหรับ session ที่ซับซ้อน (mode เดียวที่ใช้ subagent)

วิธีทำงาน (5 ขั้นตอน)

1

Gather

รัน git log + git diff --stat + ตรวจจับ session ID จาก .jsonl

2

Write Retrospective

เขียน retro ลง ψ/memory/retrospectives/YYYY-MM/DD/HH.MM_slug.md ทันที ไม่ถาม

3

Write Lesson Learned

สกัดบทเรียนลง ψ/memory/learnings/YYYY-MM-DD_slug.md

4

Oracle Sync

ส่งบทเรียนไป arra_learn() เพื่อให้ค้นหาได้ผ่าน /trace

5

Save

เขียนลง vault (ψ symlink) — ห้าม git add ψ/ เพราะเป็น shared state

Sections ที่ต้องมี

Session Summary — สรุปสั้นๆ ทำอะไรไป

Timeline — ลำดับเวลา + งานที่ทำ

Files Modified — ไฟล์ที่แก้ไข

AI Diary (150+ words) — เขียนในมุมมอง AI, vulnerable, first-person, ห้ามเขียนแบบ corporate

Honest Feedback (100+ words, 3 จุด) — 3 friction points ที่เจอจริง ห้ามเกรงใจ

Lessons Learned — บทเรียนที่ได้จาก session นี้

Next Steps — งานที่ต้องทำต่อ
