/release
Utility

Automated release flow — bump version, changelog, tag, push, GitHub release

Description

End-to-end automated release pipeline. Reads commit history since the last tag, generates a changelog, bumps the version (patch/minor/major), commits, tags, pushes to origin, and creates a GitHub release with the changelog body. Follows semver and the Oracle "Nothing is Deleted" principle — no force pushes.

Usage

/release /release --major

Modes

Default (patch)

Bump patch version (1.2.3 → 1.2.4), auto-generate changelog from commits

--minor

Bump minor version (1.2.x → 1.3.0)

--major

Bump major version (1.x.x → 2.0.0) — prompts for confirmation

--dry-run

Preview changelog and version without committing anything

วิธีทำงาน

1

Read git log since last tag

2

Generate changelog grouped by feat/fix/chore

3

Bump version in package.json / CLAUDE.md

4

Commit changelog + version bump

5

Tag + push + create GitHub release via gh CLI

Output

Git tag, GitHub Release URL, CHANGELOG.md update, version bump commit

ใช้เมื่อไหร่?

เมื่อ feature/fix batch พร้อม ship, ต้องการ GitHub Release page, หรือ deploy requires a tagged version

ใช้เมื่อไหร่ไม่ได้?

ห้ามใช้ถ้า branch ยังไม่ merge เข้า main หรือ tests ยังไม่ผ่าน — release ควรมาหลัง Ember QA เสมอ
