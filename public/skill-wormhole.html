/wormhole
Communication

Federated query proxy — ask questions across oracle nodes without moving data

Description

Send a query through the Oracle federation without physically moving data or entering a remote session. Wormhole routes your question to one or more remote oracle nodes, collects their responses via maw-js federation channels, and aggregates the answers locally. Data stays on each node — only the answer travels.

Usage

/wormhole "query" /wormhole --node pimpims-imac "what projects are active?"

Modes

Default (broadcast)

Send query to all online federation nodes, aggregate responses

--node <target>

Direct query to a specific oracle node only

--timeout <s>

Set response timeout (default 30s)

--raw

Return raw JSON responses without aggregation

วิธีทำงาน

1

Encode query as federation message via maw-js

2

Route to target node(s) through federation channels

3

Remote oracle processes query locally

4

Response travels back through wormhole

5

Aggregate and display answers with node attribution

Output

Aggregated responses from oracle nodes, each labeled with source node name and response time

ใช้เมื่อไหร่?

ต้องการถามข้อมูลจาก oracle อื่น โดยไม่ต้องเข้า SSH session, cross-oracle research, หรือ broadcast question ทั้ง federation

ใช้เมื่อไหร่ไม่ได้?

ถ้าต้องการทำงานบน remote machine โดยตรง (รัน commands, edit files) ใช้ /warp แทน — wormhole เป็น query-only
