/harden
Core

Audit Oracle configuration for safety, governance, and hardening

Description

Run a comprehensive security and governance audit on the current Oracle's configuration. Checks for exposed secrets, unsafe permissions, missing CLAUDE.md rules, governance gaps, and configuration drift against the Oracle standard. Produces a scored report with actionable fix items.

Usage

/harden /harden --deep

Modes

Default

Surface scan — CLAUDE.md rules, .gitignore, permissions, Golden Rules

--deep

Full audit — git history scan, credential patterns, hook integrity, skill trust

วิธีทำงาน

1

Read CLAUDE.md — check Golden Rules presence

2

Scan .gitignore for secret file patterns

3

Check permissions and tool allowlists

4

Audit skill configs and hook integrity

5

Generate scored report with severity levels

Output

Harden report: score (0-100), PASS/FAIL per check, severity (critical/warn/info), fix recommendations

ใช้เมื่อไหร่?

ก่อน release, หลัง onboard Oracle ใหม่, เมื่อสงสัย config drift, หรือ periodic security review ทุก sprint

ใช้เมื่อไหร่ไม่ได้?

ไม่ใช่ penetration test ระบบ production — เป็น config audit เท่านั้น ไม่ scan network vulnerabilities
