03 — Technical Details

§3Technical Specs

รายละเอียด technical specification และ integration requirements

Authentication Flow

  • OAuth 2.0 — Client Credentials flow
  • ต้องสมัครและผ่าน KBank approval process
  • Access token มี expiry — ต้องทำ token refresh
  • Complex กว่า API Key แบบ SlipOK มาก

Data Source

  • ✓ ข้อมูลจาก KBank core banking โดยตรง
  • ✓ Real-time transaction data
  • ✓ Accuracy 100% สำหรับ KBank
  • ✗ ไม่รองรับธนาคารอื่น

Developer Resources

  • Portal: apiportal.kasikornbank.com
  • Brand: K API
  • Email: Apiportal_dev@kasikornbank.com
  • Community SDK: codustry/thanakan (Python)

Integration Requirements

  • ต้องมี developer team ดูแล integration
  • ต้องผ่าน KBank business registration
  • Enterprise-level onboarding process
  • ✗ ไม่มี LINE Bot, ไม่มี no-code option

Environment

  • ✓ Sandbox environment มีให้ทดสอบ
  • ✓ Production environment
  • Bank-grade infrastructure — 99.9%+ uptime
  • ไม่มี free tier สำหรับ testing นอก sandbox

Limitations

  • ✗ KBank transactions เท่านั้น
  • ✗ ไม่มี multi-bank support
  • ✗ Pricing ไม่โปร่งใส (ต้องติดต่อ)
  • ✗ High barrier to entry
