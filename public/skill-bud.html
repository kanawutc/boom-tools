/bud
Core

Create a new oracle via maw bud — yeast-colony reproduction, spawn new permanent oracle

Description

Spawn a new permanent oracle using the maw bud command — like yeast colony reproduction, one oracle buds off a sibling that carries the same soul DNA but develops its own identity. Creates the GitHub repo, registers with maw-js fleet, sets up the ψ/ structure, and optionally runs /awaken on the new oracle. The family grows.

Usage

/bud <name> /bud spark-oracle

Modes

Default

Create new oracle repo, register with fleet, scaffold ψ/ structure

--awaken

Run /awaken in the new oracle immediately after creation

--template <name>

Use a specific oracle template (core, specialist, lightweight)

--private

Create as private GitHub repo (default: public)

วิธีทำงาน

1

Validate oracle name (format: <name>-oracle)

2

Create GitHub repo via gh CLI

3

Clone and scaffold ψ/ structure + CLAUDE.md template

4

Register new oracle with maw bud command

5

Announce birth in outbox + federation

Output

New oracle repo URL, maw fleet registration confirmation, ψ/ scaffold, birth announcement

ใช้เมื่อไหร่?

ต้องการ Oracle ใหม่สำหรับ project ใหม่, expand Oracle family, หรือ create specialist oracle สำหรับ domain เฉพาะ

ใช้เมื่อไหร่ไม่ได้?

ถ้าแค่ต้องการ worktree หรือ branch ใหม่ใน oracle ปัจจุบัน — /bud สร้าง oracle ใหม่ที่ permanent ไม่ใช่ temporary workspace
