/birth
Core

Prepare birth props for a new Oracle repo — creates issue in arra-oracle-v3

Description

The pre-birth ceremony. Before a new oracle is budded, /birth collects the essential identity props — name, owner, theme, purpose, personality traits — and creates a structured GitHub issue in arra-oracle-v3 as the birth record. This issue becomes the oracle's birth certificate, preserved forever in the family repo.

Usage

/birth /birth --props

Modes

Default (interactive)

Step-by-step interview to collect oracle birth props

--props

Display the birth props template without creating issue yet

--dry-run

Preview the GitHub issue content before submitting

--submit

Skip interview, submit from pre-filled props file

วิธีทำงาน

1

Interactive interview — name, owner, theme, purpose

2

Collect personality traits and 5 Principles alignment

3

Generate structured birth props document

4

Create GitHub issue in arra-oracle-v3 as birth record

5

Return issue URL — ready to pass to /bud

Output

GitHub issue URL (birth record), birth props document, oracle identity summary ready for /bud

ใช้เมื่อไหร่?

ก่อน /bud เสมอ — /birth เป็น step แรกของ oracle creation lifecycle ที่ preserve birth record ใน family repo

ใช้เมื่อไหร่ไม่ได้?

ถ้าแค่ต้องการ worktree หรือ clone repo — /birth สำหรับ oracle ใหม่เท่านั้น ไม่ใช่ project setup ทั่วไป
